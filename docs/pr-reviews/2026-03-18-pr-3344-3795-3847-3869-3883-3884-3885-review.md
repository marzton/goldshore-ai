# Review: PRs 3344, 3795, 3847, 3869, 3883, 3884, and 3885

Date: 2026-03-18

## Scope

Reviewed the GitHub `Files changed` view (and corresponding patch payloads) for PRs 3344, 3795, 3847, 3869, 3883, 3884, and 3885 to determine whether any branch contains a unique favicon or layout change worth preserving.

## PR-to-PR comparison

### PRs 3344 and 3795

These two PRs are effectively identical.

They both:

- replace `apps/admin/public/favicon.svg` with a Penrose gradient favicon,
- replace `apps/web/public/favicon.svg` with the same Penrose gradient favicon,
- remove the inline `ThemeManager.init()` script from `apps/admin/src/layouts/AdminLayout.astro`, and
- add inline PNG data-URI favicon / apple-touch-icon links to both admin and web layouts.

### PRs 3847, 3883, 3884, and 3885

These four PRs are effectively identical to each other.

They all:

- replace `apps/admin/public/favicon.svg` with the Penrose gradient favicon,
- replace `apps/web/public/favicon.svg` with the same Penrose gradient favicon,
- remove the inline `ThemeManager.init()` script from `apps/admin/src/layouts/AdminLayout.astro`, and
- make only a whitespace change in `apps/web/src/layouts/WebLayout.astro`.

Compared with PRs 3344/3795, these later four PRs **drop** the inline PNG data-URI favicon additions.

### PR 3869

PR 3869 starts from the same change set as PRs 3847/3883/3884/3885, then adds one extra commit that deletes `apps/admin/public/favicon.svg` entirely.

That deletion is unique among the seven PRs and would be incompatible with an admin layout that still links to `/favicon.svg`.

## Comparison to the current repository tree

### Current canonical paths

The current repository has already moved to canonical `gs-*` app paths for the active admin and web apps:

- `apps/gs-admin`
- `apps/gs-web`

So every reviewed PR is modifying older paths (`apps/admin` and `apps/web`) instead of the current canonical admin app path, and only the legacy web app path.

### Current favicon / logo state

Current tree status:

- `apps/gs-admin/src/layouts/AdminLayout.astro` already uses the Penrose logo for the admin shell (`/logo/gs-penrose.svg`), but still points its favicon to `/favicon.svg`.
- `apps/gs-admin/public/favicon.svg` is still the older monochrome glyph, not the Penrose gradient favicon proposed by the PRs.
- `packages/brand/favicon.svg` is also still the older monochrome glyph.
- `packages/theme/assets/penrose-logo.svg` already exists, confirming the Penrose logo direction has landed at the theme/brand asset level.
- `apps/gs-web/src/layouts/WebLayout.astro` already points both SVG and PNG favicon links at Penrose logo assets under `/logo/gs-penrose.*`.

## Recommendation

### Archival / closure decision

Do **not** keep any of the seven PR branches alive.

Reasoning:

1. The branches are mostly duplicate or near-duplicate.
2. The only meaningful branch-level differences are not strong reasons to preserve one:
   - PRs 3344/3795 add inline PNG data URIs, but that work targets old app paths.
   - PR 3869 deletes `apps/admin/public/favicon.svg`, which would be unsafe to carry forward as-is.
3. The current repo has already adopted the Penrose logo direction in canonical assets and in `gs-web`, but the favicon work that still matters now belongs on the **current** paths (`apps/gs-admin` and `packages/brand`), not on the stale branches behind these PRs.

### Disposition

- Close **3344, 3795, 3847, 3869, 3883, 3884, and 3885** as duplicate / superseded / obsolete.
- Record that the branding direction partially landed elsewhere (canonical Penrose logo assets and `gs-web` favicon usage), while any remaining favicon follow-up should be re-opened as a fresh PR against:
  - `apps/gs-admin/public/favicon.svg`
  - `apps/gs-admin/src/layouts/AdminLayout.astro`
  - `packages/brand/favicon.svg`
  - optionally any shared/canonical logo asset path needed for consistency
