# Gateway Routing

`gs-gateway` is the public ingress worker for GoldShore edge traffic.
`agent.goldshore.ai` is intentionally owned by the gateway and routed internally
to downstream services (including `gs-agent`) when needed.

## Routing Rules

Routes are defined in the gateway configuration and matched against the incoming request path.

| Hostname | Entrypoint | Downstream |
|------|---------|---------|
| `gateway.goldshore.ai` | `gs-gateway` | `gs-api`, `gs-agent`, internal handlers |
| `agent.goldshore.ai` | `gs-gateway` (alias ingress) | `gs-agent` |
| `gw.goldshore.ai` | `gs-gateway` | `gs-api`, `gs-agent`, internal handlers |

## Agent Ingress Model

- **Gateway-owned hostname:** `agent.goldshore.ai`
- **Entrypoint worker:** `gs-gateway`
- **Downstream service:** `gs-agent`

This avoids split hostname ownership and keeps auth, policy, and observability
enforced at a single ingress layer.

## Load Balancing

The gateway automatically load balances requests across available instances.
